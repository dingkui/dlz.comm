# 4.5 性能优化技巧

[← 返回文档导航](../00-文档导航.md)

---

## 📖 性能概述

JSONMap 底层基于 HashMap，性能与原生 Map 相当。但在某些场景下，合理使用可以进一步提升性能。

---

## 🚀 优化技巧

### 1. 复用对象

```java
// ❌ 不推荐：频繁创建对象
for (String item : items) {
    JSONMap map = new JSONMap(item);
    process(map);
}

// ✅ 推荐：复用对象
JSONMap map = new JSONMap();
for (String item : items) {
    map.clear();
    map.putAll(parseJson(item));
    process(map);
}
```

### 2. 批量操作

```java
// ❌ 不推荐：多次单个操作
JSONMap map = new JSONMap();
map.put("a", 1);
map.put("b", 2);
map.put("c", 3);

// ✅ 推荐：批量操作
Map<String, Object> data = new HashMap<>();
data.put("a", 1);
data.put("b", 2);
data.put("c", 3);
JSONMap map = new JSONMap();
map.putAll(data);
```

### 3. 选择合适的方法

```java
// ❌ 不推荐：不确定类型时用 get
Object value = map.get("key");
if (value instanceof String) {
    String str = (String) value;
}

// ✅ 推荐：直接用对应的 get 方法
String str = map.getStr("key");
```

### 4. 避免深拷贝

```java
// getMap() 返回的是引用，不是副本
JSONMap user = data.getMap("user");
user.put("age", 30);  // 会修改原对象

// 如果需要副本，手动复制
JSONMap userCopy = new JSONMap(user.toString());
```

### 5. 路径缓存

```java
// 路径解析会缓存，重复访问很快
for (int i = 0; i < 10000; i++) {
    String name = data.getStr("user.profile.name");  // 第一次解析，后续使用缓存
}
```

## 💡 性能对比

### 场景1：深层取值

```java
// 传统方式：每次都要判空和强转
long start = System.currentTimeMillis();
for (int i = 0; i < 100000; i++) {
    String city = null;
    if (data != null && data.get("user") != null) {
        Map user = (Map) data.get("user");
        if (user.get("profile") != null) {
            Map profile = (Map) user.get("profile");
            city = (String) profile.get("city");
        }
    }
}
long time1 = System.currentTimeMillis() - start;

// JSONMap 方式：路径缓存，性能更好
start = System.currentTimeMillis();
JSONMap jsonMap = new JSONMap(data);
for (int i = 0; i < 100000; i++) {
    String city = jsonMap.getStr("user.profile.city");
}
long time2 = System.currentTimeMillis() - start;

// time2 < time1（JSONMap 更快）
```

### 场景2：类型转换

```java
// 传统方式
long start = System.currentTimeMillis();
for (int i = 0; i < 100000; i++) {
    Object ageObj = data.get("age");
    Integer age = null;
    if (ageObj instanceof String) {
        age = Integer.parseInt((String) ageObj);
    } else if (ageObj instanceof Integer) {
        age = (Integer) ageObj;
    }
}
long time1 = System.currentTimeMillis() - start;

// JSONMap 方式
start = System.currentTimeMillis();
JSONMap jsonMap = new JSONMap(data);
for (int i = 0; i < 100000; i++) {
    Integer age = jsonMap.getInt("age");
}
long time2 = System.currentTimeMillis() - start;

// time2 ≈ time1（性能相当）
```

## ⚠️ 避免的坑

### 1. 不要在循环中创建 JSONMap

```java
// ❌ 错误：每次循环都创建新对象
for (String json : jsonList) {
    JSONMap map = new JSONMap(json);  // 性能差
    process(map);
}

// ✅ 正确：复用对象或批量处理
List<JSONMap> maps = jsonList.stream()
    .map(JSONMap::new)
    .collect(Collectors.toList());
```

### 2. 不要频繁序列化

```java
// ❌ 错误：频繁调用 toString()
for (int i = 0; i < 1000; i++) {
    String json = map.toString();  // 每次都序列化
    sendToApi(json);
}

// ✅ 正确：序列化一次
String json = map.toString();
for (int i = 0; i < 1000; i++) {
    sendToApi(json);
}
```

### 3. 不要过度使用路径表达式

```java
// ❌ 不推荐：多次访问同一对象
String name = data.getStr("user.profile.name");
String email = data.getStr("user.profile.email");
String phone = data.getStr("user.profile.phone");

// ✅ 推荐：先获取子对象
JSONMap profile = data.getMap("user.profile");
String name = profile.getStr("name");
String email = profile.getStr("email");
String phone = profile.getStr("phone");
```

## 📊 性能测试数据

### 测试环境
- CPU: Intel i7-10700
- 内存: 16GB
- JDK: 1.8.0_301

### 测试结果（10万次操作）

| 操作 | 传统方式 | JSONMap | 性能对比 |
|------|---------|---------|---------|
| 简单取值 | 15ms | 16ms | 相当 |
| 深层取值 | 45ms | 28ms | 快 60% |
| 类型转换 | 32ms | 30ms | 相当 |
| 构建对象 | 25ms | 30ms | 慢 20% |

**结论**：
- 深层取值：JSONMap 明显更快（路径缓存）
- 类型转换：性能相当
- 构建对象：略慢（自动创建中间层有开销）
- 整体性能：完全满足日常使用

详细测试报告：[7.1 性能测试报告](../第07章-附录/7.1-性能测试报告.md)

## 💡 最佳实践

### 1. 合理选择数据结构

```java
// 简单场景：用 Map
Map<String, Object> simple = new HashMap<>();

// 复杂场景：用 JSONMap
JSONMap complex = new JSONMap();
```

### 2. 预估容量

```java
// 如果知道大概大小，预估容量
JSONMap map = new JSONMap(100);  // 初始容量 100
```

### 3. 及时清理

```java
// 不再使用时清理
map.clear();
map = null;
```
