## 📖 安装配置问题

### Q: 如何安装 dlz.comm？

A: 在 pom.xml 中添加依赖：
```xml
<dependency>
    <groupId>top.dlzio</groupId>
    <artifactId>dlz.comm</artifactId>
    <version>6.6.3</version>
</dependency>
```

### Q: 我的项目没有 Jackson 怎么办？

A: 添加 dlz.comm 会自动引入 Jackson 依赖，无需额外配置。

### Q: 支持哪些 JDK 版本？

A: 支持 JDK 8、11、17、21 等所有主流版本。

## 📖 使用问题

### Q: JSONMap 和普通 HashMap 有什么区别？

A: JSONMap 继承自 HashMap，完全兼容 Map 接口。额外提供：
- 深层路径访问
- 自动类型转换
- 链式构建
- 智能数组处理

### Q: 路径访问失败会抛异常吗？

A: 不会。路径中任意一环为 null，直接返回 null，绝无 NPE。

### Q: put 和 set 有什么区别？

A: 
- `put("a.b", 1)` → `{"a.b": 1}`（key 不解析）
- `set("a.b", 1)` → `{"a": {"b": 1}}`（key 解析为路径）

### Q: 如何处理复杂 SQL？

A: 使用路径表达式或原生 SQL：
```java
// 路径表达式
String city = data.getStr("user.profile.addresses[0].city");

// 原生 SQL
data.sql("SELECT * FROM ...", params);
```

## 📖 性能问题

### Q: 性能怎么样？

A: 底层就是 HashMap，性能与原生 Map 一致。路径解析有缓存优化。

详见：[7.1 性能测试报告](7.1-性能测试报告.md)

### Q: 会不会有内存泄漏？

A: 不会。JSONMap 使用标准的 Java 集合，由 GC 自动管理。

---

## 📖 兼容性问题

### Q: Spring Boot 2.x / 3.x 都支持吗？

A: 都支持。dlz.comm 不依赖 Spring，只依赖 Jackson。

### Q: 可以和 FastJSON 一起用吗？

A: 可以。JSONMap 基于 Jackson 构建，与其他 JSON 库无冲突。

---

## 📖 其他问题

### Q: 支持哪些 JSON 格式？

A: 支持标准 JSON、简化 JSON（key 不加引号）、带注释的 JSON。

### Q: 线程安全吗？

A: JSONMap 本身不是线程安全的。多线程环境下需要外部同步。
