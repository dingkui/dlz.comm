## 📖 JSONMap 是什么？

JSONMap 是一个增强版的 `HashMap`，专门为 JSON 数据处理而设计。它不仅保留了 Map 的所有功能，还提供了：

- 🎯 **深层路径访问**：一行代码穿透任意层级
- 🏗️ **智能层级构建**：自动创建中间对象
- 🧬 **全能类型转换**：自动转换数据类型
- 🔗 **链式操作**：流畅的 API 设计

---

## 🎯 解决什么问题？

### 问题1：判空地狱

**传统方式：**
```java
String city = null;
if (data != null) {
    if (data.get("user") != null) {
        Map user = (Map) data.get("user");
        if (user.get("profile") != null) {
            Map profile = (Map) user.get("profile");
            city = (String) profile.get("city");
        }
    }
}
```

**JSONMap 方式：**
```java
String city = new JSONMap(data).getStr("user.profile.city");
```

### 问题2：类型强转

**传统方式：**
```java
Map<String, Object> data = getData();
Integer age = null;
Object ageObj = data.get("age");
if (ageObj instanceof String) {
    age = Integer.parseInt((String) ageObj);
} else if (ageObj instanceof Integer) {
    age = (Integer) ageObj;
} else if (ageObj instanceof Long) {
    age = ((Long) ageObj).intValue();
}
```

**JSONMap 方式：**
```java
Integer age = new JSONMap(data).getInt("age");
```

### 问题3：构建复杂结构

**传统方式：**
```java
Map<String, Object> config = new HashMap<>();
Map<String, Object> server = new HashMap<>();
server.put("host", "localhost");
server.put("port", 8080);
config.put("server", server);

Map<String, Object> db = new HashMap<>();
Map<String, Object> master = new HashMap<>();
master.put("url", "jdbc:mysql://...");
db.put("master", master);
config.put("db", db);
```

**JSONMap 方式：**
```java
JSONMap config = new JSONMap()
    .set("server.host", "localhost")
    .set("server.port", 8080)
    .set("db.master.url", "jdbc:mysql://...");
```

## 💡 核心设计理念

### 1. 有界宽容原则

JSONMap 遵循"有界宽容原则"：

- ✅ **缺失容忍**：路径不存在时返回 null，而不是抛出异常
- ✅ **空值安全**：自动处理 null 值，避免 NPE
- ✅ **类型容忍**：类型不同但能转换（"1" → 数字）
- ❌ **内容不容忍**：内容无法转换（"abc" → 数字）时抛异常

```java
JSONMap data = new JSONMap();

// 路径不存在，返回 null，不抛异常
String name = data.getStr("user.name");  // → null

// 能转换的情况：成功转换
data.put("age", "25");
Integer age = data.getInt("age");  // → 25 ✅

// 不能转换的情况：抛出异常
data.put("age", "abc");
Integer age = data.getInt("age");  // → 抛出异常 ❌

// 提供默认值（推荐）
Integer age = data.getInt("age", 0);  // 转换失败时返回 0
```

**为什么这样设计？**

- **缺失容忍**：数据结构可能不完整，路径不存在是正常情况
- **类型容忍**：前端传参类型混乱，能转换就转换
- **内容不容忍**：内容错误通常是程序 bug，应该及时发现

详见 [4.4 有界宽容原则](../第04章-高级特性/4.4-有界宽容原则.md)

### 2. 零侵入设计

JSONMap 继承自 `HashMap`，完全兼容 Map 接口：

```java
// 可以当普通 Map 使用
Map<String, Object> map = new JSONMap();
map.put("key", "value");
Object value = map.get("key");

// 也可以使用增强功能
JSONMap jsonMap = (JSONMap) map;
String str = jsonMap.getStr("key");
```

### 3. 链式操作

所有修改方法都返回 `this`，支持链式调用：

```java
JSONMap data = new JSONMap()
    .set("a", 1)
    .set("b", 2)
    .set("c", 3)
    .add("tags", "tag1")
    .add("tags", "tag2");
```

## 🔍 核心能力详解

### 能力1：深层路径访问

**路径语法：**

| 语法 | 说明 | 示例 |
|------|------|------|
| `.` | 访问对象属性 | `user.name` |
| `[n]` | 访问数组索引 | `users[0]` |
| `[-n]` | 倒数索引 | `users[-1]` |
| 混合 | 任意组合 | `users[0].profile.tags[-1]` |

**示例：**

```java
JSONMap data = new JSONMap("{\"a\":{\"b\":{\"c\":[1,2,3]}}}");

data.getInt("a.b.c[0]");     // → 1
data.getInt("a.b.c[-1]");    // → 3
data.getList("a.b.c");       // → [1,2,3]
data.getMap("a.b");          // → {"c":[1,2,3]}
```

### 能力2：智能层级构建

**自动创建规则：**

- 遇到 `.` 创建 Map
- 遇到 `[n]` 创建 List
- 自动补齐中间元素

**示例：**

```java
JSONMap json = new JSONMap();

// 自动创建嵌套对象
json.set("a.b.c", 1);
// 结果：{"a":{"b":{"c":1}}}

// 自动创建数组
json.set("arr[0]", "a");
json.set("arr[1]", "b");
// 结果：{"arr":["a","b"]}

// 自动补齐
json.set("arr[5]", "f");
// 结果：{"arr":["a","b",null,null,null,"f"]}
```

### 能力3：全能类型转换

**支持的转换：**

```java
// 字符串 → 数字
data.put("age", "25");
data.getInt("age");          // → 25

// 数字 → 字符串
data.put("count", 100);
data.getStr("count");        // → "100"

// 字符串 → 布尔值
data.put("active", "true");
data.getBoolean("active");   // → true

// 逗号分隔 → 数组
data.put("ids", "1,2,3");
data.getList("ids", Integer.class);  // → [1,2,3]

// Bean 转换
User user = data.as(User.class);
```

## 🆚 与其他库的对比

### vs 原生 Map

| 特性 | 原生 Map | JSONMap |
|------|---------|---------|
| 基本操作 | ✅ | ✅ |
| 深层访问 | ❌ 需要层层 get | ✅ 一行路径 |
| 类型转换 | ❌ 需要手动强转 | ✅ 自动转换 |
| 空值安全 | ❌ 需要判空 | ✅ 自动处理 |
| 链式操作 | ❌ | ✅ |

### vs Jackson ObjectNode

| 特性 | ObjectNode | JSONMap |
|------|-----------|---------|
| JSON 解析 | ✅ | ✅ |
| 深层访问 | ⚠️ 需要链式 get | ✅ 路径表达式 |
| 负索引 | ❌ | ✅ |
| 自动构建 | ❌ | ✅ |
| 学习成本 | 高 | 低 |

### vs FastJSON JSONObject

| 特性 | JSONObject | JSONMap |
|------|-----------|---------|
| JSON 解析 | ✅ | ✅ |
| 深层访问 | ⚠️ 需要链式 get | ✅ 路径表达式 |
| 类型转换 | ✅ | ✅ |
| 负索引 | ❌ | ✅ |
| 依赖 | FastJSON | Jackson |

## 🎯 适用场景

### ✅ 推荐使用

- API 响应解析（特别是嵌套深的）
- 动态构建请求体
- 配置文件处理
- 表单数据处理
- 数据转换和清洗
- 临时数据结构构建

### ⚠️ 谨慎使用

- 性能极致要求的场景（虽然性能已经很好）
- 需要严格类型检查的场景
- 数据结构固定且简单的场景

## 💡 最佳实践

### 1. 优先使用路径表达式

```java
// ✅ 推荐
String name = data.getStr("user.profile.name");

// ❌ 不推荐
JSONMap user = data.getMap("user");
JSONMap profile = user.getMap("profile");
String name = profile.getStr("name");
```

### 2. 提供默认值

```java
// ✅ 推荐
int age = data.getInt("user.age", 0);

// ❌ 不推荐
Integer age = data.getInt("user.age");
if (age == null) {
    age = 0;
}
```

### 3. 使用链式操作

```java
// ✅ 推荐
JSONMap request = new JSONMap()
    .set("header.token", token)
    .set("body.data", data);

// ❌ 不推荐
JSONMap request = new JSONMap();
request.set("header.token", token);
request.set("body.data", data);
```

### 4. 区分 put 和 set

```java
// put：key 不解析，直接作为键名
json.put("a.b", 1);      // {"a.b": 1}

// set：key 解析为路径
json.set("a.b", 1);      // {"a": {"b": 1}}
```
