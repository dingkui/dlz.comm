# 1.4 与其他库的对比

[← 返回文档导航](../00-文档导航.md)

---

## 📊 对比概览

| 库 | 定位 | 核心优势 | 学习成本 | 依赖 |
|---|------|---------|---------|------|
| **JSONMap** | JSON 数据操作 | 深层路径、智能构建、类型转换 | 低 | Jackson |
| Jackson | JSON 序列化 | 性能好、功能全 | 中 | 无 |
| FastJSON | JSON 序列化 | 性能极致 | 低 | 无 |
| Gson | JSON 序列化 | 简单易用 | 低 | 无 |
| Hutool JSONUtil | 工具集 | 功能丰富 | 中 | Hutool |
| JSONPath | JSON 查询 | 强大查询语法 | 高 | 无 |

---

## 🆚 vs Jackson

### Jackson 的定位

Jackson 是 Java 生态中最流行的 JSON 序列化库，专注于 JSON ↔ Java 对象的转换。

### 对比

| 特性 | Jackson | JSONMap |
|------|---------|---------|
| JSON 解析 | ✅ 强大 | ✅ 基于 Jackson |
| 序列化 | ✅ 核心功能 | ✅ 继承 Jackson |
| 深层取值 | ⚠️ 需要 `at("/a/b/c")` | ✅ `getStr("a.b.c")` |
| 负索引 | ❌ | ✅ `arr[-1]` |
| 自动构建 | ❌ | ✅ 自动创建中间层 |
| 类型转换 | ⚠️ 需要指定类型 | ✅ 自动转换 |
| 学习成本 | 中 | 低 |

### 代码对比

**场景：获取嵌套数据**

```java
// Jackson 方式
ObjectMapper mapper = new ObjectMapper();
JsonNode root = mapper.readTree(json);
String city = root.at("/user/profile/addresses/0/city").asText();

// JSONMap 方式
String city = new JSONMap(json).getStr("user.profile.addresses[0].city");
```

### 结论

- **Jackson**：负责 JSON ↔ 字符串的"搬运"
- **JSONMap**：负责拿到数据后的"加工"
- **完美互补**：JSONMap 基于 Jackson 构建，两者可以一起使用

---

## 🆚 vs FastJSON

### FastJSON 的定位

FastJSON 是阿里开源的 JSON 库，以性能著称。

### 对比

| 特性 | FastJSON | JSONMap |
|------|---------|---------|
| 性能 | ✅ 极致 | ✅ 优秀 |
| 深层取值 | ⚠️ 链式 `.getJSONObject().get()` | ✅ 路径表达式 |
| 负索引 | ❌ | ✅ |
| 自动构建 | ❌ | ✅ |
| 安全性 | ⚠️ 历史漏洞 | ✅ 基于 Jackson |

### 代码对比

```java
// FastJSON 方式
JSONObject json = JSON.parseObject(jsonStr);
String city = json.getJSONObject("user")
                  .getJSONObject("profile")
                  .getJSONArray("addresses")
                  .getJSONObject(0)
                  .getString("city");

// JSONMap 方式
String city = new JSONMap(jsonStr).getStr("user.profile.addresses[0].city");
```

### 结论

- FastJSON 性能更好，但代码冗长
- JSONMap 代码简洁，性能也足够好
- 如果项目已用 FastJSON，可以共存

---

## 🆚 vs Hutool JSONUtil

### Hutool 的定位

Hutool 是一个 Java 工具集，包含 JSONUtil 等多个模块。

### 对比

| 特性 | Hutool JSONUtil | JSONMap |
|------|----------------|---------|
| 功能范围 | 工具集（大而全） | 专注 JSON |
| 深层取值 | ⚠️ `getByPath()` | ✅ 所有方法支持路径 |
| 负索引 | ❌ | ✅ |
| 自动构建 | ❌ | ✅ |
| 包大小 | 大（整个 Hutool） | 小（~100KB） |

### 代码对比

```java
// Hutool 方式
JSONObject json = JSONUtil.parseObj(jsonStr);
Object city = JSONUtil.getByPath(json, "user.profile.addresses[0].city");

// JSONMap 方式
String city = new JSONMap(jsonStr).getStr("user.profile.addresses[0].city");
```

### 结论

- Hutool 功能全面，但体积大
- JSONMap 更专注，体积小
- 如果只需要 JSON 处理，JSONMap 更合适

---

## 🆚 vs JSONPath

### JSONPath 的定位

JSONPath 是一个 JSON 查询语言，类似 XPath。

### 对比

| 特性 | JSONPath | JSONMap |
|------|---------|---------|
| 查询能力 | ✅ 强大（支持过滤、函数） | ⚠️ 基础路径访问 |
| 学习成本 | 高（需要学习语法） | 低（直观） |
| 写入能力 | ❌ 只读 | ✅ 读写 |
| 构建能力 | ❌ | ✅ |

### 代码对比

```java
// JSONPath 方式（复杂查询）
List<String> names = JsonPath.read(json, "$.users[?(@.age > 25)].name");

// JSONMap 方式（简单路径）
String name = new JSONMap(json).getStr("users[0].name");
```

### 结论

- JSONPath 适合复杂查询
- JSONMap 适合日常数据操作
- 两者定位不同，可以共存

## 📊 性能对比

### 测试场景

- 解析 JSON 字符串
- 深层取值（5层嵌套）
- 构建复杂结构
- 类型转换

### 测试结果

| 操作 | Jackson | FastJSON | JSONMap | 说明 |
|------|---------|----------|---------|------|
| 解析 | 100% | 120% | 105% | JSONMap 基于 Jackson |
| 深层取值 | 80% | 90% | 100% | JSONMap 路径缓存优化 |
| 构建 | 100% | 110% | 95% | JSONMap 自动创建有开销 |
| 转换 | 100% | 105% | 100% | 基本一致 |

**结论**：性能与主流库相当，日常使用完全够用。

详细测试报告：[7.1 性能测试报告](../第07章-附录/7.1-性能测试报告.md)

## 🎯 选择建议

### 选择 JSONMap 的理由

✅ **代码简洁**：20 行代码缩减到 1 行  
✅ **学习成本低**：5 分钟上手  
✅ **零侵入**：继承 HashMap，完全兼容  
✅ **轻量级**：只有 ~100KB  
✅ **AI 友好**：Token 消耗降低 90%  

### 不选择 JSONMap 的理由

❌ **需要复杂查询**：用 JSONPath  
❌ **性能极致要求**：用 FastJSON  
❌ **只需要序列化**：用 Jackson 就够了

## 💡 组合使用

JSONMap 可以与其他库组合使用：

```java
// 与 Jackson 组合
ObjectMapper mapper = new ObjectMapper();
JsonNode node = mapper.readTree(json);
JSONMap map = new JSONMap(node);

// 与 FastJSON 组合
com.alibaba.fastjson.JSONObject fastJson = JSON.parseObject(json);
JSONMap map = new JSONMap(fastJson.toJSONString());

// 与 Hutool 组合
cn.hutool.json.JSONObject hutoolJson = JSONUtil.parseObj(json);
JSONMap map = new JSONMap(hutoolJson.toString());
```
